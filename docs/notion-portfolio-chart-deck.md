# Chart Deck

## 1. 프로젝트 소개

Chart Deck은 CSV를 브라우저에 바로 넣으면, 데이터 구조를 해석하고 여러 차트 후보를 비교할 수 있는 **로컬 우선 데이터 시각화 워크벤치**입니다.

사용자가 차트 종류를 먼저 고민하기보다  
**데이터를 먼저 넣고, 표현 방식과 집계 기준을 바꿔 보며 빠르게 판단할 수 있는 경험**을 목표로 만들었습니다.

브라우저에서 CSV / TSV 데이터를 로컬로 읽고, 데이터 타입을 추론한 뒤 여러 차트 후보를 비교할 수 있습니다. 서버 업로드 없이 동작하며, 샘플 데이터, 파일 선택, CSV 붙여넣기 방식을 지원합니다.

### 핵심 기능
- 샘플 데이터 / CSV 파일 / 텍스트 붙여넣기 입력
- 컬럼 타입 추론
- 차트 후보 비교
- 값 컬럼 선택
- 기준 컬럼 선택
- 날짜 기준 원본 / 일별 / 월별 / 연도별 보기
- 합계 / 평균 집계 선택
- Recharts Brush 기반 구간 탐색
- 라이트 / 다크 모드

### 차트 종류
- 선 차트
- 영역 차트
- 막대 차트
- 산점도
- 파이 차트
- 도넛 차트
- 레이더 차트

---

## 2. 어떻게 개발했는지

이 프로젝트는 **AI를 작업 가속 도구로 활용하고, 하네스와 Human-in-the-loop 방식으로 결과를 검증하며 다듬은 작업**입니다.

### 개발 방식
- AI로 요구사항, 구조 후보, 구현 초안을 빠르게 탐색
- 하네스 기반 규칙으로 테스트, 문서, 커밋, 검증 흐름 유지
- 사람이 직접 화면 밀도, 문구, 상호작용이 과한지 판단하며 정리

### 사용 스택
- React
- TypeScript
- Vite
- Recharts
- Playwright
- Vitest
- ESLint
- CSS

### 검증 흐름
- `npm test`
- `npm run typecheck`
- `npm run lint`
- `npm run build`
- `npm run e2e`
- `npm run snapshots`

### 개발에서 중요하게 본 기준
- 처음 봐도 어떤 기능인지 바로 이해되는지
- 차트가 많아져도 UI가 산만해지지 않는지
- 데이터가 없을 때와 있을 때 구조가 자연스럽게 나뉘는지
- 대량 데이터에서도 조작 방식이 복잡하지 않은지

## 3. 문제점과 해결

### 문제 1. 차트 기능이 늘수록 UI가 복잡해짐
기능이 늘어날수록 화면의 정보 밀도가 높아져, 사용자가 어디서부터 봐야 하는지 직관적으로 파악하기 어려웠습니다.

### 해결
- 의미 없는 문구 제거
- 빈 상태와 분석 상태 분리
- 입력, 차트, 해석 패널의 역할 정리
- 추천처럼 보이는 표현을 줄이고, 사용자가 직접 후보를 고르는 구조로 조정

---

### 문제 2. 직접 구현한 차트 상호작용이 조잡해짐
처음에는 SVG 기반 차트를 직접 구현하고, 그 위에 줌, 드래그, 범위 표시를 붙였습니다. 기능은 늘었지만 버튼을 켜고 끄는 흐름과 오버레이가 많아져 사용성이 어색해졌습니다.

### 해결
- 직접 구현한 확대 오버레이와 휠 줌 토글 제거
- 차트 렌더링을 Recharts 기반으로 전환
- Tooltip, ResponsiveContainer, Brush 같은 차트 기본 상호작용은 라이브러리에 위임
- 앱 UI에는 기준 컬럼, 값 컬럼, 날짜 집계처럼 사용자가 실제로 판단해야 하는 선택만 남김

---

### 문제 3. 날짜 데이터가 있을 때 특정 해석을 강제할 수 있었음
처음에는 날짜 컬럼이 있으면 일·월·년 집계를 바로 적용하는 흐름이 강했습니다. 하지만 실제 데이터에서는 원본 행 단위 흐름을 봐야 할 때도 있고, 월별 합계나 평균처럼 다른 기준이 필요할 때도 있습니다.

### 해결
- 원본 순서를 기본값으로 유지
- 막대 차트에서도 날짜 기준 월별 보기로 전환 가능
- 일별 / 월별 / 연도별 보기 옵션 추가
- 합계 / 평균 집계 선택 추가
- 숫자 컬럼이 여러 개일 때 값 컬럼을 직접 선택할 수 있게 개선

---

### 문제 4. 대량 데이터에서 조작 방식이 산만했음
처음에는 전체 보기, 최근 보기, 상위 필터, 스크롤 줌처럼 여러 조작을 붙였지만, 사용자가 어떤 버튼을 눌러야 하는지 흐름이 불명확했습니다.

### 해결
- 필터 버튼 제거
- 전체 데이터 기준 유지
- 5,000행 제한 안에서는 전체 지점을 그대로 렌더링
- 선/영역/막대 차트는 Recharts Brush로 범위를 좁혀 탐색
- 파이/도넛 차트는 비중 라벨이 겹치지 않도록 범위 확대를 적용하지 않음

---

### 문제 5. AI로 빠르게 수정할수록 구조가 과해질 수 있었음
AI로 여러 결과를 빠르게 만들 수 있었지만, 그대로 쌓이면 오히려 화면이 더 어색해질 수 있었습니다.

### 해결
- 중간 결과를 그대로 확정하지 않음
- 사람이 직접 “이 구조가 과한지” 다시 판단
- 기능보다 사용 흐름을 먼저 보고 불필요한 버튼과 설명을 제거
- 더 단순하고 목적이 분명한 구조로 재정리

결과적으로 구현 속도는 AI로 끌어올리고,  
최종 구조와 완성도 판단은 Human-in-the-loop 방식으로 마감했습니다.

---

## 짧은 소개 문안

> Chart Deck은 CSV를 브라우저에 바로 넣고, 데이터 구조를 해석해 여러 차트 후보와 집계 기준을 비교할 수 있는 로컬 우선 데이터 시각화 워크벤치입니다.  
> AI와 하네스 기반 개발 흐름으로 빠르게 구현하고, Human-in-the-loop 방식으로 UI 밀도와 상호작용을 다듬었습니다.
