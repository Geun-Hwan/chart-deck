# Chart Deck

## 1. 프로젝트 소개

Chart Deck은 CSV를 브라우저에 바로 넣으면, 데이터 구조를 해석하고 여러 차트 후보를 추천해 주는 **로컬 우선 데이터 시각화 워크벤치**입니다.

이 프로젝트는 단순히 차트를 그리는 것보다,

- 데이터를 넣고
- 어떤 차트가 잘 맞는지 바로 보고
- 다른 차트와 비교하고
- 데이터 해석 근거까지 같이 확인하는

흐름을 자연스럽게 만드는 데 집중했습니다.

즉, 사용자가 차트 종류를 먼저 고민하기보다  
**데이터를 먼저 넣고, 추천된 차트를 보며 빠르게 판단할 수 있는 경험**을 목표로 만든 프로젝트입니다.

### 핵심 기능
- 샘플 데이터 / CSV 파일 / 텍스트 붙여넣기 입력
- 컬럼 타입 추론
- 차트 후보 추천 및 비교
- 대량 데이터 확대/축소
- 라이트 / 다크 모드

### 차트 종류
- 선 차트
- 영역 차트
- 막대 차트
- 산점도
- 파이 차트
- 도넛 차트
- 레이더 차트

---

## 2. 어떻게 개발했는지

이 프로젝트는 **AI를 작업 가속 도구로 활용하고, 하네스와 사람의 판단으로 품질을 다듬는 방식**으로 개발했습니다.

### 개발 방식
- AI로 초안과 구조 후보를 빠르게 탐색
- 하네스 기반 규칙과 검증 절차 유지
- 사람이 직접 구조와 카피를 다시 판단하며 정리

### 사용한 방식
- React + TypeScript 기반 구현
- 순수 CSS와 SVG 차트 렌더링
- 테스트와 검증 반복
  - `npm test`
  - `npm run typecheck`
  - `npm run lint`
  - `npm run build`
  - `npm run e2e`
  - `npm run snapshots`

### 개발에서 중요하게 본 기준
- 처음 봐도 어떤 기능인지 바로 이해되는지
- 차트가 많아져도 UI가 산만해지지 않는지
- 데이터가 없을 때와 있을 때 구조가 자연스럽게 나뉘는지
- 대량 데이터에서도 조작 방식이 복잡하지 않은지

---

## 3. 문제점과 해결

### 문제 1. 차트 기능이 늘수록 UI가 복잡해짐
차트 종류와 상호작용이 늘어나면서 화면이 실험용 도구처럼 보일 위험이 있었습니다.

### 해결
- 의미 없는 문구 제거
- 빈 상태와 분석 상태를 분리
- 입력, 차트, 해석 패널의 역할을 분명하게 정리

---

### 문제 2. 대량 데이터에서 조작 방식이 산만했음
처음에는 버튼형 필터가 있었지만, 흐름이 끊기고 의미가 약했습니다.

### 해결
- 필터 버튼 제거
- 전체 데이터 기준 유지
- 휠 확대/축소
- 드래그 확대
- 더블클릭 초기화

결과적으로 하나의 상호작용 모델로 정리했습니다.

---

### 문제 3. AI로 빠르게 수정할수록 구조가 과해질 수 있었음
AI로 여러 결과를 빠르게 만들 수 있었지만, 그대로 쌓이면 오히려 화면이 더 어색해질 수 있었습니다.

### 해결
- 중간 결과를 그대로 확정하지 않음
- 사람이 직접 “이 구조가 과한지” 다시 판단
- 더 단순하고 목적이 분명한 구조로 재정리

결과적으로 구현 속도는 AI로 끌어올리고,  
최종 구조와 완성도 판단은 사람이 직접 정리하는 방식으로 마감했습니다.

---

## 짧은 소개 문안

> Chart Deck은 CSV를 브라우저에 바로 넣고, 데이터 구조를 해석해 적합한 차트를 추천하고 비교할 수 있는 로컬 우선 데이터 시각화 워크벤치입니다.  
> 사용자가 차트 종류를 먼저 고르지 않아도, 데이터를 넣은 뒤 추천 차트를 보며 빠르게 비교하고 해석할 수 있는 흐름을 만드는 데 집중했습니다.
